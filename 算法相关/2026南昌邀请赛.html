1. I搞快点搞快点

题目意思

长度为 lll 的赛道，从起点走到终点，每次走一步，需要花费代价为 a[i]a[i]a[i] ，但是可以使用至多 333 次技能，每次技能可以直接不花代价从当前位置 xxx 跳到 min(x+d,l)min(x + d, l)min(x+d,l) 位置，问走到终点的最小代价为多少。

题目意思大致就是这样，有改动，但是解题思路是一样的。属于签到题，直接 dpdpdp 就做完了。

参考代码

#include <bits/stdc++.h>
#define int long long
constexpr int inf = 1e18;

signed main() {
    int l, d, t0, t1, t2;
    std::cin >> l >> d >> t0 >> t1 >> t2;
    std::string s;
    std::cin >> s;
    std::vector<int> a(l + 1);
    for (int i = 1; i <= l; i++) {
        if (s[i - 1] == '0') a[i] = t0;
        else if (s[i - 1] == '1') a[i] = t1;
        else a[i] = t2;
    }
    std::vector dp(l + 1, std::vector<int>(4, inf));
    dp[0][0] = 0;
    for (int i = 1; i <= l; i++) {
        for (int j = 0; j <= 3; j++) {
            dp[i][j] = std::min(dp[i][j], dp[i - 1][j] + a[i]);
            if (i - d >= 0 && j) {
                dp[i][j] = std::min(dp[i][j], dp[i -d][j - 1]);
            }
        }
    }
    int ans = std::min({dp[l][0], dp[l][1], dp[l][2], dp[l][3]});
    for (int i = l - d; i <= l; i++) {
        ans = std::min({ans, dp[i][0], dp[i][1], dp[i][2]});
    }
    std::cout << ans << "\n";
    return 0;
}

2. H粑粑博弈

题目意思

依旧 AliceAliceAlice 和 BobBobBob 玩一个游戏，AliceAliceAlice 先手。有 nnn 堆石子，给定一个 kkk，每次可以选择一堆石子取走 xxx 个石子，其中 x≤kx \le kx≤k 并且 xxx 小于该堆石子的个数，取完后下一个玩家取的石子个数必须小于等于上一个玩家取的石子个数。当有个人无法继续取石子时，该人输掉游戏。给定初始每堆石子的数量，问谁能赢。

解题思路

首先我们考虑一堆石子的情况，假设这堆石子数量为 xxx ：

  1. 当 k=1k = 1k=1 时，只需要判断石子数量的奇偶性即可，奇数时 AliceAliceAlice 赢，偶数时 BobBobBob 赢。
  2. 当 k=2k = 2k=2 时，奇数时先手只需要选择取 111 个石子，则回到上一个情况的状态，故先手必胜，进一步的，无论 kkk 为多少，奇数时先手必胜，偶数时先手必败。故我们下面都只需考虑偶数情况。当为偶数时，没人愿意先取 111 个石子，因为这样就会打破奇偶从而导致对手必胜。故最开始每个人都是取 222 个石子，直到最后只剩一个或两个石子。但是不可能为一个，因为我们现在的石子数量是偶数。从而现在是看 x2\frac{x}{2}2x​ 的奇偶性，如果为奇数，先手必胜，否则必败。
  3. 当 k=3k = 3k=3 时，由上面结论只分析偶数情况，先手第一次肯定不会取三个石子，因为这样打破奇偶导致对手必胜，故先手依旧取 222 个石子，此时问题就变成了 k=2k = 2k=2 的情况，进一步的，我们分析可知只需讨论 kkk 为偶数的情况。
  4. 当 k=4k = 4k=4 时，又分两种情况，当 x2\frac{x}{2}2x​ 为奇数时，回到上面的分析知先手必胜。当 x2\frac{x}{2}2x​ 为偶数时，先手第一次肯地不会取 222 个石子，因为取两个就是回到上面分析的导致对手必胜，那么只能取 444 个石子，巧妙的地方来了，取四个并不会改变 x2\frac{x}{2}2x​ 的奇偶性，故后手回到先手的情况，如果他足够聪明也会取 444 个石子，那么他们又这样持续下去直到最后只剩 444 个 （因为 x2\frac{x}{2}2x​ 为偶数所以肯定可以被 444 整除），此时就是分析 x4\frac{x}{4}4x​ 的奇偶性，如果为奇数，先手必胜，否则必败。
  5. 当 kkk 为 666 时，还是前面分析的，有一种情况为奇数，先手必胜，如果所有情况都为偶数，即到 x4\frac{x}{4}4x​ 都还是偶数，那么我们来分析一下，先手肯定只能取 666 个石子，其他的情况上面都有分析会必败。但是这里有意思了，因为 6/2=36 / 2 = 36/2=3 ，取完后剩余石子除以2肯定为奇数为先手必胜，即对手必胜。

分析了那么多，其实已经很明显了，就是看是 x/2x / 2x/2 ，x/4x / 4x/4 ... 到除以最大的不超过 kkk 的二的幂是否有奇数，有的话先手必胜，否则必败。

那么有多堆石子呢？看每堆石子这样除的和是否有奇数即可。仔细想一下，是可以想通的，我这里懒得写了。有一个点我说一下，就是比如说每次每人都取 444 个石子的时候，有没有可能某一次某个人取 222 个能改变他原来取 444 的命运？不可能的，如果可以改变，那么先手第一步直接取 222 就好了，没必要取着取着就变，哎表述不清楚自己理解吧。

3. C旅行者的愿望（金牌题）

题目意思

懒得说了，看题目链接吧。

解题思路

事实上就是一个矩阵快速幂的改编，只要对矩阵理解足够深刻，对矩阵的乘法理解足够深刻，那么很容易解决这题。这里就是把矩阵乘法换成另一个能恰好满足题目要求的一种操作，需要满足结合律，这样才能顺理成章使用快速幂。难就难在这种转换上，具体看代码。定义新的矩阵操作也可以解决很多问题，比如全源最短路问题，但时间复杂度达到 O(n3logn)O(n^3logn)O(n3logn)，并没有比Floyd算法更优。

参考代码

#include <bits/stdc++.h>
#define int long long
constexpr int inf = 1e9;

struct Mat {
    int n;
    std::vector<std::vector<int>> a;
    explicit Mat(const int _n) : n(_n), a(n + 1, std::vector<int>(n + 1, inf)) {}
    Mat operator*(const Mat& mat) const {
        Mat res(n);
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= n; j++) {
                for (int k = 1; k <= n; k++) {
                     res.a[i][j] = std::min(res.a[i][j], std::max(a[i][k], mat.a[k][j]));
                }
            }
        }
        return res;
    }

    [[nodiscard]] Mat pow(int k) const {
        Mat res(n);
        Mat x = *this;
        for (int i = 1; i <= n; i++) res.a[i][i] = 0;
        while (k) {
            if (k & 1) res = res * x;
            x = x * x;
            k >>= 1;
        }
        return res;
    }
};

signed main() {
    std::ios::sync_with_stdio(false), std::cin.tie(nullptr), std::cout.tie(nullptr);
    int n, m;
    std::cin >> n >> m;
    Mat a(n);
    for (int i = 1, u, v; i <= m; i++) {
        std::cin >> u >> v;
        a.a[u][v] = 0;
    }
    int k;
    std::cin >> k;
    for (int i = 1, u, v; i <= k; i++) {
        std::cin >> u >> v;
        a.a[u][v] = std::min(a.a[u][v], i);
    }
    int q, w;
    std::cin >> q >> w;
    const auto res = a.pow(w);
    for (int i = 1; i <= q; i++) {
        int s, t;
        std::cin >> s >> t;
        if (res.a[s][t] < inf) std::cout << res.a[s][t] << "\n";
        else std::cout << "-1\n";
    }
    return 0;
}

4. JGCD 之旅（金牌题）

题目意思

懒得说了，看题目链接吧。

解题思路

从起点到终点，要使经过的边的最小值尽可能大，那么我们尽量走大一点的边，于是可以想到从大到小枚举边权，把有这条边的两点连起来，然后直到起点到终点可达，那最后加的这条边就是要求的。这不就是最大生成树吗！有多次询问，那么只需要在最大生成树上倍增查询就行了。

参考代码

#include <bits/stdc++.h>
#define int long long
constexpr int N = 2e6 + 10;

struct DSU {
    int n;
    std::vector<int> fa;
    explicit DSU(const int _n) : n(_n), fa(n + 1) {
        std::iota(fa.begin(), fa.end(), 0);
    }

    int find(const int x) {
        if (x == fa[x]) return x;
        return fa[x] = find(fa[x]);
    }
};

signed main() {
    std::ios::sync_with_stdio(false), std::cin.tie(nullptr), std::cout.tie(nullptr);
    int n, q;
    std::cin >> n >> q;
    std::vector<int> a(n + 1);
    std::vector<int> vis(N);
    DSU dsu(N);
    std::vector<int> node(N), fa(N), deep(N);
    std::vector up(N, std::vector<int>(22));
    for (int i = 1; i <= n; i++) std::cin >> a[i], vis[a[i]] = 1;
    const int mx = *std::max_element(a.begin() + 1, a.end());
    int cnt = mx;
    for (int i = mx; i >= 1; i--) {
        int fst = -1;
        for (int j = i; j <= mx; j += i) {
            if (!vis[j]) continue;
            if (fst == -1) fst = j;
            else {
                const int fax = dsu.find(fst);
                if (const int fay = dsu.find(j); fax != fay) {
                    ++cnt;
                    dsu.fa[fax] = cnt;
                    dsu.fa[fay] = cnt;
                    node[cnt] = i;
                    fa[fax] = cnt;
                    fa[fay] = cnt;
                }
            }
        }
    }
    for (int i = cnt; i >= 1; i--) {
        if (fa[i] == 0) {
            up[i][0] = i;
            deep[i] = 0;
        } else {
            deep[i] = deep[fa[i]] + 1;
            up[i][0] = fa[i];
        }
        for (int j = 1; j <= 21; j++) {
            up[i][j] = up[up[i][j - 1]][j - 1];
        }
    }
    while (q--) {
        int x, y;
        std::cin >> x >> y;
        x = a[x], y = a[y];
        if (x == y) std::cout << x << "\n";
        else {
            if (deep[x] < deep[y]) std::swap(x, y);
            const int deep_delta = deep[x] - deep[y];
            for (int i = 0; i <= 21; i++) {
                if (deep_delta >> i & 1) x = up[x][i];
            }
            int lca;
            if (x == y) lca = x;
            else {
                for (int i = 21; i >= 0; i--) {
                    if (up[x][i] != up[y][i]) {
                        x = up[x][i];
                        y = up[y][i];
                    }
                }
                lca = up[x][0];
            }
            std::cout << node[lca] << "\n";
        }
    }
    return 0;
}

5. G世界的意义（金牌题）

题目意思

懒得说了，看题目链接吧。

解题思路

数据结构题，强如Gemini也写了三遍才AC，直接给代码吧。

参考代码

#include <iostream>
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

const int MAXN = 200005;

struct Node {
    int a;       // 未匹配的 ')' 数量
    int b;       // 未匹配的 '(' 数量
    int len;     // 区间长度
    int ans;     // 区间内最长合法连续子串长度
    bool is_leaf;
    char c;
} tree[MAXN * 4];

string s;

// 获取 u 节点消耗最后 k 个 '(' 后，连同前面的有效括号构成的后缀长度
int query_suff(int u, int k) {
    if (tree[u].is_leaf) {
        if (tree[u].c == ')') return 0;
        return k == 1 ? 1 : 0;
    }
    int lc = 2 * u, rc = 2 * u + 1;

    // 如果需要的 '(' 数量比右孩子少，完全在右侧解决
    if (k < tree[rc].b) {
        return query_suff(rc, k);
    }
    // 如果相等，且右侧自带的 ')' 超出了左侧能提供的 '('，导致无法继续向左穿透
    else if (k == tree[rc].b && tree[rc].a > tree[lc].b) {
        return query_suff(rc, k);
    }
    // 向左穿透：吃满右侧，并将剩下的需求和右侧的内部需求丢给左侧
    else {
        return tree[rc].len + query_suff(lc, tree[rc].a + k - tree[rc].b);
    }
}

// 获取 u 节点消耗最前 k 个 ')' 后，连同后面的有效括号构成的前缀长度
int query_pref(int u, int k) {
    if (tree[u].is_leaf) {
        if (tree[u].c == '(') return 0;
        return k == 1 ? 1 : 0;
    }
    int lc = 2 * u, rc = 2 * u + 1;

    // 逻辑与 query_suff 完全对称
    if (k < tree[lc].a) {
        return query_pref(lc, k);
    } else if (k == tree[lc].a && tree[lc].b > tree[rc].a) {
        return query_pref(lc, k);
    } else {
        return tree[lc].len + query_pref(rc, tree[lc].b + k - tree[lc].a);
    }
}

// 向上合并逻辑
void push_up(int u) {
    int lc = 2 * u, rc = 2 * u + 1;
    tree[u].len = tree[lc].len + tree[rc].len;
    tree[u].a = tree[lc].a + max(0, tree[rc].a - tree[lc].b);
    tree[u].b = tree[rc].b + max(0, tree[lc].b - tree[rc].a);

    int k = min(tree[lc].b, tree[rc].a);
    int boundary = query_suff(lc, k) + query_pref(rc, k);
    tree[u].ans = max({tree[lc].ans, tree[rc].ans, boundary});
}

// 建树
void build(int u, int start, int end) {
    if (start == end) {
        tree[u].is_leaf = true;
        tree[u].c = s[start];
        tree[u].len = 1;
        tree[u].ans = 0;
        if (s[start] == '(') {
            tree[u].a = 0; tree[u].b = 1;
        } else {
            tree[u].a = 1; tree[u].b = 0;
        }
        return;
    }
    int mid = start + (end - start) / 2;
    build(2 * u, start, mid);
    build(2 * u + 1, mid + 1, end);
    tree[u].is_leaf = false;
    push_up(u);
}

// 单点修改翻转
void update(int u, int start, int end, int pos) {
    if (start == end) {
        if (tree[u].c == '(') {
            tree[u].c = ')';
            tree[u].a = 1; tree[u].b = 0;
        } else {
            tree[u].c = '(';
            tree[u].a = 0; tree[u].b = 1;
        }
        return;
    }
    int mid = start + (end - start) / 2;
    if (pos <= mid) update(2 * u, start, mid, pos);
    else update(2 * u + 1, mid + 1, end, pos);
    push_up(u);
}

// 用于收集查询覆盖到的线段树节点
struct QueryNode {
    int a, b, len, ans, node_idx;
};
vector<QueryNode> q_nodes;

// 收集区间
void collect_nodes(int u, int start, int end, int l, int r) {
    if (l <= start && end <= r) {
        q_nodes.push_back({tree[u].a, tree[u].b, tree[u].len, tree[u].ans, u});
        return;
    }
    int mid = start + (end - start) / 2;
    if (l <= mid) collect_nodes(2 * u, start, mid, l, r);
    if (r > mid) collect_nodes(2 * u + 1, mid + 1, end, l, r);
}

// 供区间查询使用的状态记录
struct State {
    int a, b, len, ans;
};

// 严谨的区间查询与合并
int solve_query(int l, int r, int n) {
    q_nodes.clear();
    collect_nodes(1, 1, n, l, r);
    if (q_nodes.empty()) return 0;

    // 动态维护前缀的状态
    vector<State> prefs(q_nodes.size());
    prefs[0] = {q_nodes[0].a, q_nodes[0].b, q_nodes[0].len, q_nodes[0].ans};

    for (size_t i = 1; i < q_nodes.size(); i++) {
        int a1 = prefs[i-1].a, b1 = prefs[i-1].b;
        int a2 = q_nodes[i].a, b2 = q_nodes[i].b;

        prefs[i].a = a1 + max(0, a2 - b1);
        prefs[i].b = b2 + max(0, b1 - a2);
        prefs[i].len = prefs[i-1].len + q_nodes[i].len;

        int k = min(b1, a2);

        // 按照与 query_suff 完全相同的阻断逻辑计算左侧后缀长度
        int suff = 0;
        int cur_k = k;
        for (int j = i - 1; j >= 0; j--) {
            int u = q_nodes[j].node_idx;
            int left_b = (j > 0) ? prefs[j-1].b : 0;

            if (cur_k < tree[u].b) {
                suff += query_suff(u, cur_k);
                break;
            } else if (cur_k == tree[u].b && tree[u].a > left_b) {
                suff += query_suff(u, cur_k);
                break;
            } else {
                suff += tree[u].len;
                cur_k = tree[u].a + cur_k - tree[u].b;
            }
        }

        int pref = query_pref(q_nodes[i].node_idx, k);
        prefs[i].ans = max({prefs[i-1].ans, q_nodes[i].ans, suff + pref});
    }

    return prefs.back().ans;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    if (!(cin >> n >> q)) return 0;

    cin >> s;
    s = " " + s; // 1-based index mapping

    build(1, 1, n);

    while (q--) {
        int op;
        cin >> op;
        if (op == 1) {
            int k;
            cin >> k;
            update(1, 1, n, k);
        } else if (op == 2) {
            int l, r;
            cin >> l >> r;
            cout << solve_query(l, r, n) << "\n";
        }
    }

    return 0;
}

队友为何如此强悍？感觉他真的是热爱。我看着他一步一步超越我然后把我甩在身后好远。感觉他刷题纯粹是瘾大，我刷题是有目的的，我不是纯粹的热爱，我还有为了名利，为了得奖，为了不以后学计算机连个有含金量的奖都没有讨饭吃。我又是个三分钟热度的人，很难干成大事，做事都凭兴趣，总喜欢待在舒适圈不愿突破。

可以注意到，那些能做成大事的人，除了有天赋外，还是能持之以恒的人。一次的成功不是成功，一次的失败也不是失败，坚持才是胜利，笑到最后才是胜利。

多做些有意义的事吧，别天天刷视频了。等多年以后，千万不要又后悔大学没好好努力，就像现在后悔高中没认真学一样。命运一直都掌握在自己的手里，尽力去做，没做成也不会有遗憾。
