区间调度及其变种问题分析（2026.4.28）

在算法竞赛和实际工程中（如会议室安排、CPU 任务调度），我们经常会遇到 “区间调度（Interval Scheduling）” 问题：给定一组线段（或任务），每个都有起始点 sis_isi​ 和结束点 eie_iei​，要求从中选出互不相交的几条。

随着约束条件和优化目标的变化，这个问题的解法会从极其优雅的贪心算法，无缝切换到带有二分优化的动态规划（DP）。本文将带你一次性打通区间调度的三个经典变种！

🟢 场景一：基础版 —— 只求“选出最多”的线段

问题描述： 有多个线段，要求选择互不相交的几条，如何才能使选出的线段数量最多？

这是最经典的“活动选择问题（Activity Selection）”。目标单一，只需追求数量极致。

核心解法：贪心算法 (Greedy)

对于这种纯粹追求数量的问题，贪心策略就是全局最优解。

  1. 核心排序： 将所有线段按照结束点 (eie_iei​) 从小到大进行排序。
  2. 贪心选择： 维护一个变量 last_end 记录当前已选中线段的最晚结束时间（初始值为 −∞-\infty−∞）。
  3. 顺序遍历： 如果当前线段的起始点 si≥last_ends_i \ge \text{last\_end}si​≥last_end（说明没有重叠），就毫不犹豫地选中它！并更新 last_end=ei\text{last\_end} = e_ilast_end=ei​。

💡 为什么按结束点排序最牛？
贪心的本质是“给未来留出最多的可能性”。每次都选结束最早的线段，可以最大限度地把时间轴后面的空间空出来，从而容纳更多的后续线段。

  • 时间复杂度： O(Nlog⁡N)O(N \log N)O(NlogN)（主要耗时在排序，遍历只需 O(N)O(N)O(N)）。

🔴 场景二：加强版 —— 既要“数量最多”，又要“长度最长”

问题描述： 在保证选出线段数量最多的前提下，要求这些线段的总长度最大。

一旦引入了“长度（权重）”这一第二维度的最优条件，单纯的贪心算法就会彻底失效。我们需要请出算法界的大杀器：动态规划 (DP) 结合 二分查找。

核心解法：带二分优化的 DP

此时我们需要权衡“选与不选”的代价，状态转移是核心。

  1. 依然排序： 将所有线段按结束点从小到大排序。

  2. 状态定义： 设 dp[i]dp[i]dp[i] 表示在前 iii 条线段中进行选择，能得到的最优解。

    • 注意，这里的最优解不再是一个单纯的数字，而是一个二元组：{max_count, max_length}。

  3. 二分查找找前置： 对于第 iii 条线段（长度为 wi=ei−siw_i = e_i - s_iwi​=ei​−si​），如果我们要选它，就必须找到在它开始之前就已经结束的最后一条线段。利用二分查找（如 C++ 的 std::upper_bound），可以快速找到最大的索引 jjj，满足 ej≤sie_j \le s_iej​≤si​。

  4. 状态转移方程：
    面临第 iii 条线段，我们决策：

    • 不选： 继承前一个状态 dp[i−1]dp[i-1]dp[i−1]。
    • 选： 状态变为 dp[j]dp[j]dp[j] 加上当前线段，即 dp[j]+{1,wi}dp[j] + \{1, w_i\}dp[j]+{1,wi​}。

    dp[i]=max⁡(dp[i−1],dp[j]+{1,wi})dp[i] = \max(dp[i-1], dp[j] + \{1, w_i\}) dp[i]=max(dp[i−1],dp[j]+{1,wi​})

    (注：二元组的 max⁡\maxmax 比较逻辑是优先比较第一维 count，若相同再比较第二维 length)

  • 时间复杂度： O(Nlog⁡N)O(N \log N)O(NlogN)（排序 Nlog⁡NN \log NNlogN + 遍历时 NNN 次二分查找 Nlog⁡NN \log NNlogN）。

🟡 场景三：纯享版 —— 抛开数量，只求“总长度最长”

问题描述： 抛开线段数量的限制，只关心选中的互不相交线段的总长度（或总时间/总价值）最大。

这就是经典的 “带权区间调度（Weighted Interval Scheduling）” 问题。它其实是场景二的简化版。

核心解法：带二分优化的 DP

框架与场景二完全一致，只是我们的 dpdpdp 状态变回了简单的一维数值。

  1. 按结束点排序。
  2. 状态定义： 设 dp[i]dp[i]dp[i] 表示在前 iii 条线段中选取，能得到的最大总长度。
  3. 二分查找前置不相交线段 jjj。
  4. 状态转移方程：

    dp[i]=max⁡(dp[i−1],dp[j]+wi)dp[i] = \max(dp[i-1], dp[j] + w_i) dp[i]=max(dp[i−1],dp[j]+wi​)

    其中 wiw_iwi​ 是第 iii 条线段的长度。
  • 时间复杂度： 依然保持优雅的 O(Nlog⁡N)O(N \log N)O(NlogN)。
