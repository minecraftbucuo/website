如何求数组所有数对的 GCD 与 LCM 之和？（2026.4.28）

在算法竞赛（如 ACM/ICPC、Codeforces）中，我们可能会遇到一类经典的数论组合问题：给定一个长度为 NNN 的数组 AAA，求其中所有可能数对的两两最大公因数（gcd⁡\gcdgcd）或最小公倍数（lcm\text{lcm}lcm）之和。

即计算：

Sum=∑1≤i<j≤Ngcd⁡(A[i],A[j])\text{Sum} = \sum_{1 \le i < j \le N} \gcd(A[i], A[j]) Sum=1≤i<j≤N∑​gcd(A[i],A[j])

或

Sum=∑1≤i<j≤Nlcm(A[i],A[j])\text{Sum} = \sum_{1 \le i < j \le N} \text{lcm}(A[i], A[j]) Sum=1≤i<j≤N∑​lcm(A[i],A[j])

如果使用双层循环暴力枚举，时间复杂度是 O(N2log⁡M)O(N^2 \log M)O(N2logM)（其中 MMM 是数组最大值），当 NNN 达到 100,000 级别时必然超时。本文将带你通过数论转化与容斥 DP，将这类问题的时间复杂度降维打击至 O(Mlog⁡M)O(M \log M)O(MlogM)。

第一部分：全局 GCD 之和的两种解法

求 gcd⁡\gcdgcd 之和有两种主流解法：欧拉函数法和容斥 DP 法。掌握它们是解决更复杂变形的基础。

解法一：欧拉函数的视角转换

1. 核心数学定理
欧拉函数 φ\varphiφ 有一个绝妙的性质：一个数的所有因子的欧拉函数之和，等于该数本身。

n=∑d∣nφ(d)n = \sum_{d|n} \varphi(d) n=d∣n∑​φ(d)

将这个性质代入到我们要求的 gcd⁡\gcdgcd 中：

gcd⁡(A[i],A[j])=∑d∣A[i] 且 d∣A[j]φ(d)\gcd(A[i], A[j]) = \sum_{d | A[i] \text{ 且 } d | A[j]} \varphi(d) gcd(A[i],A[j])=d∣A[i] 且 d∣A[j]∑​φ(d)

2. 转换枚举视角：计算贡献
暴力做法是“看每对数字有哪些公共因子”。我们将其转换为 “看每一个可能的因子 ddd，它在所有数字对中出现了多少次”。

设 cnt[d]cnt[d]cnt[d] 表示数组 AAA 中，是 ddd 的倍数的元素的个数。
从这 cnt[d]cnt[d]cnt[d] 个数中任选 2 个，它们的公因数必然包含 ddd。选法共有 C(cnt[d],2)C(cnt[d], 2)C(cnt[d],2) 种。
因此，总和公式化简为：

Sum=∑d=1Mφ(d)×cnt[d]×(cnt[d]−1)2\text{Sum} = \sum_{d=1}^{M} \varphi(d) \times \frac{cnt[d] \times (cnt[d] - 1)}{2} Sum=d=1∑M​φ(d)×2cnt[d]×(cnt[d]−1)​

3. 算法实现

  • 使用线性筛在 O(M)O(M)O(M) 内预处理出所有的 φ(d)\varphi(d)φ(d)。
  • 统计原数组元素的词频 freq。
  • 枚举 ddd，内部枚举 ddd 的倍数计算 cnt[d]cnt[d]cnt[d]，再累加答案。复杂度为经典的调和级数求和 O(Mlog⁡M)O(M \log M)O(MlogM)。

解法二：容斥 DP（莫比乌斯反演的容斥形式）

相比于欧拉函数，容斥 DP 的思想更加直白，且更容易扩展（比如扩展到求 lcm\text{lcm}lcm）。

1. 状态定义：从“包含”到“恰好”

  • F[d]F[d]F[d]：选出的两个数，公因数包含 ddd（即都是 ddd 的倍数）的数对数量。
  • f[d]f[d]f[d]：选出的两个数，最大公因数恰好等于 ddd 的数对数量。

2. 状态转移
计算 F[d]F[d]F[d] 非常简单，和前面一样：

F[d]=cnt[d]×(cnt[d]−1)2F[d] = \frac{cnt[d] \times (cnt[d] - 1)}{2} F[d]=2cnt[d]×(cnt[d]−1)​

F[d]F[d]F[d] 包含了最大公因数是 d,2d,3d…d, 2d, 3d \dotsd,2d,3d… 的情况。我们要剥离出恰好是 ddd 的情况，只需将其减去：

f[d]=F[d]−∑k=2⌊M/d⌋f[k×d]f[d] = F[d] - \sum_{k=2}^{\lfloor M/d \rfloor} f[k \times d] f[d]=F[d]−k=2∑⌊M/d⌋​f[k×d]

注意遍历顺序： 为了保证减去的 f[k×d]f[k \times d]f[k×d] 都是已经计算好的正确值，我们需要从大到小倒序枚举 ddd（从 MMM 递减到 1）。最后累加 ∑(d×f[d])\sum (d \times f[d])∑(d×f[d]) 即为答案。

第二部分：进阶挑战 —— 全局 LCM 之和

如果将问题换成求 lcm\text{lcm}lcm，欧拉函数法就无能为力了，但容斥 DP依然可以完美绝杀！

1. 破题点
利用公式 lcm(x,y)=x×ygcd⁡(x,y)\text{lcm}(x, y) = \frac{x \times y}{\gcd(x, y)}lcm(x,y)=gcd(x,y)x×y​。
我们可以依然按 gcd⁡=d\gcd=dgcd=d 对数对进行分组。如果我们能求出 “最大公因数恰好为 ddd 的所有数对的乘积之和”（设为 f[d]f[d]f[d]），那么它们对 lcm\text{lcm}lcm 最终答案的贡献就是 f[d]d\frac{f[d]}{d}df[d]​。

2. 状态定义转换

  • G[d]G[d]G[d]：公因数包含 ddd 的所有数对的乘积之和。
  • f[d]f[d]f[d]：最大公因数恰好等于 ddd 的所有数对的乘积之和。

3. O(1) 计算两两乘积之和的数学魔法
在求 G[d]G[d]G[d] 时，我们需要把所有 ddd 的倍数两两相乘再求和。如果用双重循环会退化成 O(N2)O(N^2)O(N2)。这里我们可以借用完全平方公式的推广：

💡 数学小贴士：矩阵视角下的乘积求和
对于任意一组数，它们两两组合的乘积之和等于：（整体的和的平方 - 每个数的平方和）/ 2

∑i<jXiXj=(∑Xi)2−∑(Xi2)2\sum_{i < j} X_i X_j = \frac{\left( \sum X_i \right)^2 - \sum \left( X_i^2 \right)}{2} i<j∑​Xi​Xj​=2(∑Xi​)2−∑(Xi2​)​

利用这个公式，我们只需枚举 ddd 的倍数，统计它们的一次方之和 (SdS_dSd​) 和平方和 (SqdSq_dSqd​)，就能直接算出：

G[d]=(Sd)2−Sqd2G[d] = \frac{(S_d)^2 - Sq_d}{2} G[d]=2(Sd​)2−Sqd​​

4. 容斥转移与统计
状态转移与 gcd⁡\gcdgcd 问题完全一致（倒序遍历）：

f[d]=G[d]−∑k=2⌊M/d⌋f[k×d]f[d] = G[d] - \sum_{k=2}^{\lfloor M/d \rfloor} f[k \times d] f[d]=G[d]−k=2∑⌊M/d⌋​f[k×d]

最终的 lcm\text{lcm}lcm 总和为：

Total Sum=∑d=1Mf[d]d\text{Total Sum} = \sum_{d=1}^{M} \frac{f[d]}{d} Total Sum=d=1∑M​df[d]​

⚠️ 避坑指南：数据溢出与取模运算

在实际的编程竞赛中，求 lcm\text{lcm}lcm 之和的题目 100% 会要求对一个大质数（如 1000000007 或 998244353）取模，因为数值增长极快。

涉及取模时，你必须注意以下两点：

  1. 除法变乘法： 公式中的“除以 2”必须替换为乘以 2 的乘法逆元。
  2. 分母逆元： 最后一步计算 f[d]d\frac{f[d]}{d}df[d]​ 时，必须替换为 f[d]×inv(d)(modP)f[d] \times \text{inv}(d) \pmod Pf[d]×inv(d)(modP)。
  3. 计算平方和小心溢出： 计算 (k×d)2(k \times d)^2(k×d)2 时，先将 k×dk \times dk×d 取模，然后再平方取模，防止 long long 溢出。

总结

从 gcd⁡\gcdgcd 到 lcm\text{lcm}lcm，核心思想都是利用调和级数将成对枚举 (O(N2)O(N^2)O(N2)) 转化为值域枚举 (O(Mlog⁡M)O(M \log M)O(MlogM))。容斥 DP 提供了一个极其灵活且统一的框架，结合简单的代数变形公式，就能在代码层面实现优雅的“降维打击”。
