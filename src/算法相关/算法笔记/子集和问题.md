# Top-X 子集和问题（2026.5.21）

**Top-X 子集和（Top-X Subset Sums）** 是一类经典的组合最优化问题。给定一个序列，要求按照特定规则（如限制元素个数、是否允许重复选择等）选取子集，并按总和从小到大的顺序输出前 $X$ 个合法的子集和。

通过优先队列（最小堆）维护状态机转移，可以在 $O(X \log X)$ 的时间复杂度内高效解决这类问题。本文将详细探讨三种常见的约束场景及其对应的算法模型，并给出其状态转移不重不漏的严格正确性证明。

---

### 1. 前置处理：负数的转化

在利用最小堆求解前 $X$ 小的子集和时，算法的核心前提是**状态转移必须满足单调不降**（即扩展出的新状态，其总和必定大于等于当前状态）。如果原序列中存在负数，会破坏这一性质。

对于存在负数的序列，可通过以下步骤进行 $O(n \log n)$ 的等价转化：

1. 提取序列中所有的负数，计算它们的和，记为 `base_sum`。
2. 将序列中所有的负数取绝对值，转化为非负序列。
3. 对转化后的非负序列进行升序排序。
4. 在非负序列上求解前 $X$ 个最小的子集和，对于每一个求出的和 $S$，其实际答案为 $S + base\_sum$ 。

这背后的数学逻辑是：“不选一个负数”等价于“在包含所有负数的基础和上，加上该负数的绝对值”。**下文的讨论均假设序列 $A$ 已按照非负且升序排列，即 $0 \le A_1 \le A_2 \le \dots \le A_n$。**

---

### 2. 选取小于 $K$ 个元素（不可重复）

**规则：** 从序列中选取最多 $K-1$ 个不同位置的元素。

由于序列均为非负数，绝对最小的子集和必定是**空集**，其和为 $0$。除空集外，我们可以通过维护一个二叉生成树，利用“追加”与“替换”两种操作，不重不漏地遍历所有合法子集。

我们定义状态为 `(sum, idx, cnt)`：

* `sum`：当前子集的元素总和。
* `idx`：当前子集中最后加入的元素在原序列中的下标。
* `cnt`：当前子集包含的元素个数。

初始时，将空集的和 $0$ 计入答案。随后将只包含最小元素的状态 `(A[1], 1, 1)` 压入最小堆。每次弹出堆顶状态时，进行以下两次状态转移：

1. **Append（追加）：** 若 `cnt + 1 < K` 且后续还有元素，保留当前子集，加入下一个元素。转移为 `(sum + A[idx + 1], idx + 1, cnt + 1)`。
2. **Replace（替换）：** 若后续还有元素，将当前子集末尾的元素替换为下一个元素。转移为 `(sum - A[idx] + A[idx + 1], idx + 1, cnt)`。

#### 正确性证明（不重不漏）：

要证明该转移方式能无重复地生成所有合法子集，需证明**每个合法子集在状态转移树中都有且仅有唯一的父节点（即唯一的生成路径）**。

假设我们生成了一个合法子集，观察其最后两个加入的元素下标：$i_{last}$ 和 $i_{prev}$（若仅有一个元素则 $i_{prev} = 0$）。逆推其生成过程，仅存在两种严格互斥的情况：

1. **$i_{last} = i_{prev} + 1$**：即末尾两个元素在原数组中是相邻的。这说明元素 $A[i_{last}]$ 必定是通过 **Append（追加）** 操作，附加在以 $A[i_{prev}]$ 结尾的子集之后的。
2. **$i_{last} > i_{prev} + 1$**：即末尾两个元素在原数组中不相邻，存在间隙。这说明元素 $A[i_{last}]$ 必定是通过 **Replace（替换）** 操作，将前一个状态中的 $A[i_{last} - 1]$ 替换掉而生成的。

由于这两种情况互斥且涵盖了所有可能，任何子集向上回溯的历史路径是唯一的，这就从数学上保证了状态转移树是一棵严格的树形结构，不存在多条路径到达同一子集的情况（不重），且能覆盖所有子集（不漏）。

```cpp
#include <bits/stdc++.h>
#define int long long

struct State {
    int sum, idx, cnt;
    bool operator>(const State& other) const {
        return sum > other.sum;
    }
};

signed main() {
    int n, k, x;
    std::cin >> n >> k >> x;
    std::vector<int> a(n + 1);
    for (int i = 1; i <= n; i++) std::cin >> a[i];
    std::sort(a.begin() + 1, a.end());

    std::cout << 0 << "\n"; // 空集
    if (x == 1) return 0;

    std::priority_queue<State, std::vector<State>, std::greater<State>> pq;
    pq.push({a[1], 1, 1});

    for (int step = 1; step < x; step++) {
        if (pq.empty()) break;
        auto [sum, idx, cnt] = pq.top();
        pq.pop();
        
        std::cout << sum << "\n";
        
        if (cnt + 1 < k && idx + 1 <= n) {
            pq.push({sum + a[idx + 1], idx + 1, cnt + 1});
        }
        if (idx + 1 <= n) {
            pq.push({sum - a[idx] + a[idx + 1], idx + 1, cnt});
        }
    }
    return 0;
}

```

### 3. 选取小于 $K$ 个元素（可重复选择）

**规则：** 从序列中选取最多 $K-1$ 个元素，同一个位置的元素可以被重复选取多次。这也是完全背包类组合最值的一个变种。

此时，状态 `idx` 不再表示最后选取元素的物理位置，而是表示当前正在使用的**元素种类**。
状态定义依然为 `(sum, idx, cnt)`，初始态推入 `(A[1], 1, 1)`。每次弹出堆顶时：

1. **Push（选取同类）：** 若 `cnt + 1 < K`，再加入一个与当前种类相同的元素。转移为 `(sum + A[idx], idx, cnt + 1)`。
2. **Downgrade（种类降级）：** 保持元素数量不变，将最后一个加入的元素替换为下一个种类（值更大的种类）。转移为 `(sum - A[idx] + A[idx + 1], idx + 1, cnt)`。

#### 正确性证明（不重不漏）：

由于允许元素重复，我们要避免诸如 $[A_1, A_2]$ 和 $[A_2, A_1]$ 这种组合相同但顺序不同导致的重复计算。此算法通过强制**种类编号非递减**的方式解决了该问题。

逆推任何一个按种类非递减排列的合法多重集（如包含种类 $T_{prev}$ 和 $T_{last}$，且 $T_{prev} \le T_{last}$），其父节点必然唯一：

1. **$T_{last} = T_{prev}$**：即多重集的末尾有两个相同的种类。这必然是由 **Push（选取同类）** 操作生成的。
2. **$T_{last} > T_{prev}$**：即末尾的种类发生了变化。这必然是由 **Downgrade（种类降级）** 操作，将末尾种类为 $T_{last} - 1$ 的状态替换而来的。

这种转移设计确保了同一个多重集只能按确定的单调序列生成，彻底杜绝了排列重复。

```cpp
#include <bits/stdc++.h>
#define int long long

struct State {
    int sum, idx, cnt;
    bool operator>(const State& other) const {
        return sum > other.sum;
    }
};

signed main() {
    int n, k, x;
    std::cin >> n >> k >> x;
    std::vector<int> a(n + 1);
    for (int i = 1; i <= n; i++) std::cin >> a[i];
    std::sort(a.begin() + 1, a.end());

    std::cout << 0 << "\n";
    if (x == 1) return 0;

    std::priority_queue<State, std::vector<State>, std::greater<State>> pq;
    pq.push({a[1], 1, 1});

    for (int step = 1; step < x; step++) {
        if (pq.empty()) break;
        auto [sum, idx, cnt] = pq.top();
        pq.pop();
        
        std::cout << sum << "\n";
        
        if (cnt + 1 < k) {
            pq.push({sum + a[idx], idx, cnt + 1});
        }
        if (idx + 1 <= n) {
            pq.push({sum - a[idx] + a[idx + 1], idx + 1, cnt});
        }
    }
    return 0;
}

```

### 4. 精确选取 $K$ 个元素（不可重复）

**规则：** 从序列中恰好选取 $K$ 个不同位置的元素。

这是限制最为严格的情况。最小的组合和必定是选取序列的前 $K$ 个元素（即 $A_1 \dots A_K$）。我们可以将这 $K$ 个选中的元素视作 $K$ 个令牌。为了使和变大且不产生重复组合，令牌只能向右滑动，且每个令牌不能越过其右侧相邻令牌的位置。

定义状态 `(sum, pos, idx, limit)`：

* `sum`：当前 $K$ 个元素的总和。
* `pos`：当前正在操作的令牌下标。
* `idx`：当前令牌的编号（从 $1$ 到 $K$）。
* `limit`：当前令牌向右滑动的红线（即下一个不可触碰的物理位置）。

初始时，操作最右侧的第 $K$ 号令牌，其位置为 $K$，向右滑动的上限为 $N+1$。
每次弹出堆顶时，进行两步判定：

1. **右移当前令牌：** 若当前令牌的下一步没有触碰红线 `pos + 1 < limit`，将其右移。转移为 `(sum - A[pos] + A[pos + 1], pos + 1, idx, limit)`。
2. **唤醒左侧相邻令牌：** 若当前令牌已经移动过（`pos > idx`，即腾出了它原本的位置），则冻结当前令牌，唤醒左侧编号为 `idx - 1` 的令牌，让其向右滑动一步。左侧令牌的新红线即为当前令牌的位置 `pos`。转移为 `(sum - A[idx - 1] + A[idx], idx, idx - 1, pos)`。

#### 正确性证明（不重不漏）：

该转移机制本质上是一种**严格的自右向左锁定协议**。

假设我们要证明任意合法最终分布的生成路径是唯一的。在此算法下：

1. 操作必须从最右侧的第 $K$ 号令牌开始。
2. 只有当令牌 $i$ 至少向右移动过一步（让出了初始位置）后，才有资格唤醒左侧的令牌 $i-1$。
3. **关键约束（单向控制权转移）**：一旦使用“唤醒”操作将控制权转移给左侧的令牌 $i-1$，右侧的令牌 $i$ 将被**永久冻结**，其位置在此后的该分支转移中再也无法改变。

这意味着，对于任何一个最终确定的布局，其生成的时间顺序是被唯一“写死”的：必定是最右侧的令牌先滑动到其最终目的地，然后交出控制权；接着倒数第二个令牌滑动到其最终目的地，再交出控制权，依次向左递推。
因为每个令牌到达终点后立即冻结右侧结构，使得“先移动左边再移动右边”这种打乱顺序的路径在算法中被彻底阻断，从而确保了任意一个选数分布只有一条唯一的生成轨迹。

```cpp
#include <bits/stdc++.h>
#define int long long

struct State {
    int sum, pos, idx, limit;
    bool operator>(const State& other) const {
        return sum > other.sum;
    }
};

signed main() {
    int n, k, x;
    std::cin >> n >> k >> x;
    std::vector<int> a(n + 1);
    for (int i = 1; i <= n; i++) std::cin >> a[i];
    std::sort(a.begin() + 1, a.end());

    int base_sum = 0;
    for (int i = 1; i <= k; i++) base_sum += a[i];

    std::priority_queue<State, std::vector<State>, std::greater<State>> pq;
    pq.push({base_sum, k, k, n + 1});

    for (int step = 0; step < x; step++) {
        if (pq.empty()) break;
        auto [sum, pos, idx, limit] = pq.top();
        pq.pop();
        
        std::cout << sum << "\n";
        
        if (pos + 1 < limit) {
            pq.push({sum - a[pos] + a[pos + 1], pos + 1, idx, limit});
        }
        if (idx > 1 && pos > idx) {
            pq.push({sum - a[idx - 1] + a[idx], idx, idx - 1, pos});
        }
    }
    return 0;
}

```
